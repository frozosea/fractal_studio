# M1/M2 Gap Closure

Дата закрытия: 2026-07-24.

Все gaps из первоначального аудита M1/M2 устранены в коде, миграциях и
контрактах. Этот файл остаётся как closure record, а не как список открытых
проблем.

| Former gap | Closure | Evidence |
|---|---|---|
| Platform used legacy C++ `/api/*` | Platform mapper/client now use only authenticated Compute v1: `/compute/v1/previews`, `/compute/v1/runs`, `/runs/{id}`, `/runs/{id}/cancel`, manifest and private artifact stream. Durable request uses stable `idempotencyKey=platform-job:{renderJobId}`. | `app/studio/compute_request_mapper.py`, `app/infrastructure/compute/compute_client.py` |
| Compute spec was stale | `compute-openapi.yaml` and M2/M3 sections describe current v1 envelope, `computeRunId`, manifest checksum/size and private artifact route. Legacy API is explicitly out of contract. | `docs/compute-openapi.yaml`, `docs/platform-backend-spec.md` |
| Poll consumed retry budget | `attempt_count` is delivery telemetry; new `retry_count` drives exponential backoff/dead-letter only. Normal `RescheduleOutboxEvent` leaves retry count unchanged. | migration `20260724_0005_outbox_retry_counter.py`, `app/outbox/{models,repository,worker}.py` |
| Dead event left render/quota stuck | M2 registry has per-render dead-letter handler. It locks/render-terminalizes job, releases reservation and writes system audit event. | `app/studio/render_worker.py`, `app/studio/render_job_repository.py` |
| Stale quota reservation had no owner | `RenderQuotaExpiryScheduler` emits durable expiry work; M2 handler atomically expires reservation, terminalizes job and audits it. Default worker wires scheduler. | `app/studio/quota_expiry_scheduler.py`, `app/outbox/worker.py` |
| Render never became Asset | Concrete M3 `AssetIngestionService` is default worker port. It creates processing asset, streams only selected v1 manifest artifact, verifies SHA-256/size, uploads private master, records `asset_file`, completes job and releases quota. | `app/assets/{service,repository}.py`, `app/infrastructure/{compute/compute_artifact_reader.py,storage/object_storage.py}` |
| Cancel/Compute failure race lost cancel intent | Infrastructure failure no longer transitions `cancel_requested` to `failed`; cancellation wins and reservation is released. | `app/studio/render_job_repository.py` |
| Public errors had FastAPI `detail` body | Global handlers return `{error:{code,message,details}}` for HTTP, validation and unhandled errors. | `app/main.py` |
| Logging/correlation policy absent | Middleware emits UUIDv7 request IDs, response header and correlation context. Logger writes exact UTC pipe field order locally and same structured JSON fields in production; idempotency/user ID use `-` when absent. | `app/main.py`, `app/core/{logging,request_context,access_middleware}.py` |
| Login/register had no brute-force protection | Redis fail-closed per email+IP limits protect public login/register. | `app/auth/rate_limit.py`, `app/auth/router.py` |
| Production auth cookie could be insecure | Settings rejects production config without secure cookie, strong non-dev secret, HTTPS origin and non-local CORS. | `app/core/config.py`, `tests/unit/test_gap_closures.py` |
| Julia recipe semantic hole | Canonical recipe rejects `julia=true` without complete `juliaRe/juliaIm`. | `app/studio/models.py`, `tests/unit/test_gap_closures.py` |
| Recipe pagination could emit empty next page | Query reads `limit + 1`; cursor exists only when a further row exists. | `app/studio/recipe_service.py` |
| Standard pytest command failed imports | Pytest config sets project root on `pythonpath`; no `PYTHONPATH=.` shell workaround required. | `pyproject.toml` |

## Verification

```text
uv run ruff check app tests  -> passed
uv run pytest -q             -> 20 passed, 5 skipped
alembic upgrade head         -> PostgreSQL at 20260724_0005
```

The five skipped tests are opt-in live E2E tests: they require explicit
`E2E_API_URL`, `E2E_DATABASE_URL` and (for render) `E2E_COMPUTE_AVAILABLE=1`.
Their Compute stub now implements the same private v1 paths/envelope, idempotent
submit, manifest and artifact bytes used by Platform.

Deployment acceptance remains operational work: run C++ Compute v1 with
`FSD_ENABLE_LEGACY_API=0`, set same `FSD_COMPUTE_SERVICE_KEY` / `COMPUTE_SERVICE_KEY`,
then run the opt-in live E2E suite. It is not a remaining implementation gap.
